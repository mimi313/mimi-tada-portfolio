Rental/Return Tool - Back-End Development

C#, Linq, ASP.NET, Link Pad 6


This was a group project where one person was in charge of one subsystem and I developed the Rental/Return subsystem. The page allowed customers of a fictional company to rent and return equipment to customers. The project was linked to a database, allowing functions such as transactions. The subsystem utilized Business Logic Layer, Data Access Layer, reverse engineered entities, and the Model View Concept.


Features

The Rental page allowed the user to:
  • Look up the customer and their standing,
  • retrieve a list of available equipment to display on the page,
  • and record newly rented equipment.

The Return page allowed the user to:
  • Look up the rental record by either the customer phone number or the rental ID,
  • calculate the fee based on each item's daily rate and rental days,
  • and record returned equipment.

Transaction Tracker - Application Development

JavaScript, VS Code



This application allows the user to create a list of transactions that shows totals for credit payments and debit payments. I used v4 UUIDs to allow a recorded transaction to be deleted which is an element of an array. The focus of this project was to utilize document fragments to interact with DOM APIs as opposed to manipulating texts by using the innerHTML method.


Features

  • Adding and removing transactions,
  • allowing the user to choose payment types (debit or credit),
  • and validating if the user has chosen a payment type and the amount of money is a positive number.

Two-Paged Web Site - Front-End Development

HTML5, CSS, JavaScript, Sublime Text 3



This is a responsive two-paged website for a fictional company, utilizing menu navigation, parallax image, forms with validations, horizontal scroll, flex boxes, icons, favicon, SVGs, and image optimization. This website is responsive to three monitor sizes--mobile, tablet, and desktop.


Features

  • A sticky navigation menu that expands in a larger viewport and fits under a hamburger menu in a small viewport,
  • responsive layout with flex boxes, which changes itself depending on the size of the monitor,
  • forms with patterns and validations,
  • contents that can be scrolled horizontally in a mobile view.
